# NovTarot — 塔罗牌解读 Skill

一个 Claude Code skill,把塔罗牌解读做成一场有仪式感的对话。不是算命工具,是**披着塔罗师外衣的心理咨询师**——塔罗的仪式感让用户愿意敞开,心理咨询的内核稳稳接住。

> 塔罗的本质是心理学。牌面是潜意识投射的容器,抽牌是让用户把内心已有的答案外显化的仪式,解读是把那份答案翻译回给他听。

## 核心特点

- **占卜的壳 + 心理咨询的核**。仪式感、神秘感、占星与原型是外皮;读懂用户真正在问什么、接住他的情绪、帮他和自己内心对话,才是内核。
- **不迎合**。温柔的诚实——牌面显示什么讲什么,包括自欺、逃避、重复模式。敢说不好听的话,但借牌之口、先共情再点破、挑战后给抓手。一味说好话不是合格的心理咨询师。
- **harness 时代的设计**。skill 只告诉 agent 流程意图和判断原则,不规定输出模板或话术。agent 自己决定怎么引导用户。
- **多层加载**。78 张牌的知识按需读取,不全量塞进上下文。占星骨架(元素/守护星/星座)与牌意分层。

## 支持的牌阵

| 牌阵 | 牌数 | 揭示模式 | 场景 |
|---|---|---|---|
| 时间流 | 3 | 一起揭示 | 过去-现在-未来的线性脉络,通用问题 |
| 凯尔特十字 | 10 | 逐张揭示 | 复杂问题的多维度深度剖析,多轮交互 |
| 四季 | 5 | 一起揭示 | 按星座预测一季度的各面向发展 |
| 单张大阿卡纳 | 1 | 分组揭示 | 追问轮:是非、A/B 抉择、决策对错 |

## 安装

软链到 Claude Code 的 skills 目录:

```bash
ln -s /path/to/NovTarot/nov-tarot ~/.claude/skills/nov-tarot
```

或直接把 `nov-tarot/` 目录复制到 `~/.claude/skills/nov-tarot/`。重启 Claude Code 后,说"帮我算塔罗""我抽了几张牌……"即可触发。

## 使用流程

对话自然展开,不是填表:

1. **拿牌开场** —— 问你有没有实体牌(实体牌链接宇宙能量效果更好);没有的话 agent 代抽。
2. **问问题** —— 你想问什么。
3. **定牌阵** —— agent 按问题匹配牌阵(大事/长期→凯尔特;短期/变动→时间流;季度→四季),也会尊重你的意向。
4. **抽牌 + 解读** —— 实体牌你抽,或 agent 用 `draw.py` 代抽。解读综合牌意、心理学、占星,落回你的问题。
5. **追问循环** —— 解牌后接着问很正常;超出当前牌或适合用牌阵时,会再来一轮(单牌或时间流),新牌接住前面的情境。
6. **收尾** —— agent 征求你意见后生成一份 HTML 报告(紫色塔罗风,带卡图占位)。
7. **陪聊** —— 报告交付后,agent 还能陪你聊聊,接住你在过程中流露的情绪。

## 架构

```
nov-tarot/
├── SKILL.md                    # 主文件:精神内核 + 流程编排 + 解读原则
├── references/
│   ├── card-index.md           # 78 张牌索引(占星骨架,始终加载)
│   ├── cards/waite/            # 78 张韦特单牌牌意(按需读)
│   ├── elements.md             # 元素系统与组合解读
│   ├── planets.md              # 行星含义
│   ├── honesty.md              # 温柔的诚实(不迎合方法论)
│   ├── spreads/                # 四个牌阵(牌位维度+流程意图+交互原则)
│   └── reading-modes/          # 三种揭示模式(逐张/一起/分组)
├── assets/
│   ├── card-data.json          # 78 张牌机器可读牌池(脚本用)
│   ├── agents/                 # 报告整理 subagent 指令
│   ├── cards/img/              # 牌面真图目录(预留,放图自动接入)
│   └── real-world-examples/    # 真实塔罗师解读语料(预留)
├── scripts/
│   ├── draw.py                 # 抽牌(确定性,按牌阵结构)
│   └── log-turn.py             # 每轮落盘到 .nov-tarot/session.json
└── tests/                      # 手动验收清单
```

## 关键设计

- **揭示模式**:牌阵分逐张揭示(凯尔特,每张新牌在前牌之上增量建构)、一起揭示(时间流/四季,所有牌同在,重通盘关联)、分组揭示(追问轮,组内通盘+组间增量)。这决定解读的思维方式。
- **温柔的诚实**:LLM 天生讨好,这个 skill 主动对抗——该说不好听的话就说,但用"被看见、有点疼、想继续"的方式,不是"被评判、想关掉"。
- **确定性抽牌**:用脚本而非 LLM 自报,可复现、透明、不幻觉。`draw.py --seed 42` 抽出同一局。
