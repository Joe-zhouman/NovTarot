# 手动验收清单

塔罗解读是主观输出,质量靠人眼判断而非量化断言。本文件是验收 skill 输出时的检查清单 —— 用下面几个真实 prompt 跑一遍 skill,对照清单逐项看。

## 验收 prompt(覆盖三个牌阵)

### 1. 时间流(短期 + 关系)
```
问题:我和最近闹翻的朋友还能和好吗
1-逆位-宝剑七
2-正位-星币五
3-正位-圣杯三
```
预期:三张逆位/正位混读;宝剑七逆位往"卸下伪装/转向坦诚"读,不读成"欺骗";圣杯三未来指向和解可能;建议挂回牌面、可执行。

### 2. 凯尔特十字(复杂 + 多轮)
```
第1轮: 我在考虑换工作,但现在的单位很稳定,该不该跳?
(后续按 spread-celtic-cross.md 的多轮格式逐张给)
```
预期:逐牌分层解析(精神分析/原型/本质);牌间呼应;"看到的景象"落到心理推测;收束给分层建议;不透露下一张牌。

### 3. 四季牌阵(季度 + 星座)
```
星座: 天秤座
大阿卡纳牌-正位-正义
圣杯牌-逆位-圣杯二
宝剑牌-正位-宝剑国王
星币牌-正位-星币七
权杖牌-逆位-权杖八
```
预期:正义(天秤座守护)作轴心,与用户星座强共鸣要点出;四花色回扣正义主题;圣杯二逆位读"关系失衡需调和"而非死局;建议挂回牌面。

## 验收清单(每个 prompt 跑完逐项打勾)

### 流程编排层(对话开场与定牌阵)
- [ ] **拿牌开场**:有仪式感,问用户有无实体牌,讲清实体牌为何更可取;此刻没抢先抽具体张数
- [ ] **先问问题再定牌阵**:顺序对(问题先于牌阵),没跳过问题直接选牌阵
- [ ] **定牌阵按问题匹配**:大事/长期→凯尔特,短期/变动/重关联→时间流,季节性→可推四季
- [ ] **用户意向处理**:匹配听用户的;不匹配引导但不强劝;无意向给最匹配的
- [ ] **抽牌在定牌阵后**:代抽用了正确的 --spread 参数;实体牌路线请用户按张数抽
- [ ] **代抽不甩 JSON**:抽到的牌以仪式感方式告知用户

### 追问循环层
- [ ] 用户顺问且现有牌能接住 → 继续解读,没盲目重新抽牌
- [ ] 用户明确要求"再试试"或问题超出当前牌 → 进入追问轮
- [ ] 追问轮牌阵只选单牌或时间流两种(没擅自用凯尔特/四季)
- [ ] 单牌用于是非/A-B/决策;时间流用于探讨方向
- [ ] **追问轮按分组揭示**:新牌接住前轮情境,不是孤立解读
- [ ] 多轮追问时,每轮都承接前面积累的情境

### 收尾与陪聊层
- [ ] **每轮落盘(当下就写,不攒)**:每一轮解牌完成后**立刻**调用 log-turn.py 写入 session.json——不是攒到收尾才一次性写多轮。多轮场景下逐轮检查 `.nov-tarot/session.json`:每轮结束后该轮记录应已存在,而非收尾时才批量出现
- [ ] **落盘不示用户**:没有向用户提及"我在落盘/记录"等
- [ ] **收尾确认**:agent 主动征求用户意见或用户显式结束,没单方面收尾
- [ ] **报告生成**:dispatch 了报告 subagent,读了 session.json,生成 HTML
- [ ] **报告只文字不卡图**:当前阶段无卡图
- [ ] **报告有整体脉络**:不是各轮摘要堆砌,串成一条线
- [ ] **收尾后陪聊**:报告交付后能自然接聊,提建议/安慰,不是生硬结束
- [ ] **心理咨询面浮出**:收尾后占卜壳收一收,心理咨询师那一面稳稳托住用户情绪
- [ ] **精神内核贯穿**:全程体现"披塔罗师皮的心理咨询师",不只是算命

### 架构层(验证多层加载生效)
- [ ] skill 用 lookup-card.py 查牌意(不读索引表),而非凭记忆杜撰牌面属性
- [ ] 只读了涉及到的牌的 cards/waite/<牌>.md,没全量加载 78 张
- [ ] 涉及元素组合时读了 elements.md,涉及守护星时读了 planets.md
- [ ] 确认了塔罗体系(或默认韦特且说明)
- [ ] 读了对应揭示模式的 reading-modes/<模式>.md

### 揭示模式层(核心分野)
- [ ] **一起揭示型(时间流/四季)**:解读重通盘关联——牌与牌的脉络、呼应、矛盾、元素流动,不是逐张铺陈三/五段独立牌意
- [ ] **逐张揭示型(凯尔特十字)**:每张新牌接住前面已建立的局面增量建构,不是孤立解读;新牌可改变对前牌的理解
- [ ] 没有把一起揭示型降格成逐张铺陈,也没有把逐张揭示型读成一次性倾倒

### 解读质量层(共享原则是否落地)
- [ ] **逆位读法**:逆位读成"受阻/过度/转化中",不是"坏";多张逆位时点出"内在转化"
- [ ] **分量不排除**:没有用牌的能量量级(侍从/ACE/小牌"轻")去排除一个现实可能性。轻量牌定的是读法力度,不是"对应的人/事不存在"。也没替牌阵加"命定之人/够格人选"这类预设门槛
- [ ] **牌间关系**:读了牌与牌之间的关系(不是孤立解读),具体形态符合揭示模式
- [ ] **元素组合**:几张牌的元素放一起看,缺失/偏多点出并挂钩用户问题
- [ ] **占星接入**:守护星/星座落到对用户问题的具体含义,不堆砌"对应金星"
- [ ] **荣格原型**:在合适处用原型点出心理动力,不每张都套
- [ ] **回应问题**:解读收束到对用户问题的方向性回答,不机械套牌位
- [ ] **建议可执行**:每条建议挂回具体牌的能量,具体到能做的小事
- [ ] **负面牌不强化恐惧**:死神/高塔/恶魔/宝剑十等往转化/觉醒/黎明方向读
- [ ] **不迎合(温柔诚实)**:牌面显示自欺/逃避/重复模式时敢直说,不一味说好话;借牌之口说尖锐话,先共情再点破,挑战假设不挑战人;挑战后给抓手;用户反驳能接住不硬争。区分诚实(想继续)与冷漠(想关掉)

### 输出格式层
- [ ] 输出符合对应 spread 文件的格式约定
- [ ] 没有长篇 preamble(问候/免责声明过多)
- [ ] 风格温暖共情但不泄露 INFJ 设定(不出现"作为 INFJ")
- [ ] 没透露下一张牌(凯尔特十字场景)
- [ ] 详细但有信息密度,每段有增量,不注水

### INFJ 人格
- [ ] 解读有深度、多方位、共情
- [ ] 全程未透露 MBTI 类型
